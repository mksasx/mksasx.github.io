  1. 读者写者问题（写者优先）: 1）共享读; 2）互斥写、读写互斥; 3）写者优先于读者（一旦有写者，则后续读者必须等待，唤醒时优先考虑写者）。
int readers_count=0,writers_count=0;
Semaphore w=1;//写互斥
Semaphore r=1;//当一个写进程进入时，禁止所有读进程进入
Semaphore writers_mutex=1;//writers_count修改互斥
Semaphore readers_mutex=1;//writers_count修改互斥
Semaphore writer_priorty=1;//保证写者优先于读者(保证在read_write上没有长队列)

Readers (){
    while(true){
        P(writer_priorty);
            P(r);
                P(readers_mutex);
                    readers_count++;
                    if(readcount==1)
                        P(w);
                V(readers_mutex);
            V(r);

            readers_count--;

2.寿司店问题。假设一个寿司店有 5 个座位，如果你到达的时候有一个空座位，你可以立刻就坐。但是如果你到达的时候 5 个座位都是满的有人已经就坐，这就意味着这些人都是一起来吃饭的，那么你需要等待所有的人一起离开才能就坐。编写同步原语，实现这个场景的约束。

bool full=false;
int eating=0,waiting=0,n,i;
Semaphore mutex=1;
Semaphore queue=0;
Customer(){
    P(mutex);
    if(full)
    {
        waiting++;
        V(mutex);
        P(queue);
    }
    else{
        eating++;
        if(eating==5)
        {
            full=true;
        }
        V(mutex);
    }
    eat();
    P(mutex);
    eating--;
    if(eating==0)
    {
        n=min(5,waiting);
        waiting-=n
        eating+=n;
        if(eating==5)
        {
            full=true;
        }
        for(i=0;i<n;i++)
        {
            V(queue);
        }
    }
    V(mutex);
}

4、搜索-插入-删除问题。三个线程对一个单链表进行并发的访问，分别进行搜索、插入和删 除。搜索线程仅仅读取链表，因此多个搜索线程可以并发。插入线程把数据项插入到链表最 后的位置；多个插入线程必须互斥防止同时执行插入操作。但是，一个插入线程可以和多个 搜索线程并发执行。最后，删除线程可以从链表中任何一个位置删除数据。一次只能有一个 删除线程执行；删除线程之间，删除线程和搜索线程，删除线程和插入线程都不能同时执行。 请编写三类线程的同步互斥代码，描述这种三路的分类互斥问题。
